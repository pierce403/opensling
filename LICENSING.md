# Licensing

OpenSling uses a simple split-license model so reuse is obvious:

- Documentation text, diagrams, tables, and educational media are licensed under `CC BY 4.0` unless a page says otherwise.
- Code, scripts, configuration, workflows, and fabrication design sources are licensed under `Apache-2.0` unless a file says otherwise.

Practical rule:

- if you want to copy, translate, print, remix, or republish the written documentation, you can do that under `CC BY 4.0`
- if you want to fork the code, scripts, CAD, or fabrication files, you can do that under `Apache-2.0`

Repository references:

- `LICENSE`: Apache-2.0 text for code and design-source artifacts
- `LICENSE-docs.md`: CC BY 4.0 notice for documentation content

If a stricter or different license applies to a specific asset, that asset should say so explicitly in its file header or adjacent README.
